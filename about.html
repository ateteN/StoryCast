About StoryCast

Our Mission

We believe that every person deserves access to compelling stories. StoryCast is built on three core values:

  • Transparency: We hold ourselves accountable to our readers and the truth.
  • Diversity: we actively avoid the usual perspective that dominates normal media.

Accessibility at StoryCast

StoryCast meets WCAG 2.1 Level AA accessibility standards across all pages and content formats. Below is a summary of the accessibility features that are implemented.

  • Full Text Transcripts

    Each audio story page contains a complete, timestamped text transcript. This benefits users who prefer to read or users with slow connections who cannot stream audio

  • Keyboard Navigation

    All interactive elements (navigation links, audio controls, form fields, and buttons) are fully usable with a keyboard alone (Tab, Shift+Tab, Enter, Space). A visible skip-navigation link at the top of every page allows keyboard users to jump directly to the main content.

  • High-Contrast Design

    StoryCast uses a gray-background colour scheme with high-contrast text that meets WCAG 2.1 AA and All interactive states (focus, hover, visited) are visible and distinct.

  • Semantic HTML for Screen Readers

    Every page is built with semantic HTML5 elements:
    <header>, <nav>, <main>, <article>, <section>, <figure>, and <footer>
    so that screen readers such as NVDA can describe the structure and each page to their users.

  • Descriptive Alt Text

    Every image on StoryCast includes a meaningful alt attribute that provides the information a user would get from viewing an image. Decorative images are given an empty alt="" so screen readers will skip them since they are not meaningful.

  • ARIA Labels

    Where HTML semantic elements are insufficient, we use ARIA attributes such as aria-label, aria-labelledby, aria-describedby, and aria-current to provide additional context for screen readers.

If you encounter any accessibility barrier on StoryCast, please contact us and we will resolve the issue as quickly as possible.
